# 能力管理板块验收标准（可自动化）

## 1. 发布门禁
1. P0 用例通过率 100%。
2. P1 用例通过率 >= 98%。
3. 接口契约测试全部通过（状态码、错误码、字段类型）。
4. 性能指标通过：10,000 条能力数据下，列表查询 P95 < 500ms。
5. 安全指标通过：敏感字段脱敏、内网 URL 拦截、越权拦截。

## 2. 分类管理验收项（P0）
| AC-ID | 场景 | 验收标准 |
|---|---|---|
| AC-CAT-001 | 新增分类成功 | 返回 200，列表可查询到新增分类 |
| AC-CAT-002 | 分类名必填 | 空值或全空格返回 400 + CAP_4001 |
| AC-CAT-003 | 分类名长度限制 | 11 字符返回 400 + CAP_4001；10 字符可成功 |
| AC-CAT-004 | 分类名去重规则 | 忽略大小写和连续空格后重复，返回 400 + CAP_4002 |
| AC-CAT-005 | 删除空分类 | 返回 200，列表中不存在该分类 |
| AC-CAT-006 | 删除非空分类拦截 | 返回 400 + CAP_4005 |
| AC-CAT-007 | 内置分类保护 | 编辑或删除内置分类返回拒绝（403 或业务拒绝码） |
| AC-CAT-008 | 分类排序 | 查询顺序满足 sort asc, created_at desc |

## 3. 能力管理验收项（P0）
| AC-ID | 场景 | 验收标准 |
|---|---|---|
| AC-CAP-001 | 新增外部应用能力 | 返回 200，数据落库，列表可查 |
| AC-CAP-002 | 新增外部工作流能力 | 返回 200，数据落库，列表可查 |
| AC-CAP-003 | 禁止新增同步类型 | 新增问学应用/问学工作流返回 400 + CAP_4001 |
| AC-CAP-004 | 能力名必填和长度限制 | 空值或 >10 字符返回 400 + CAP_4001 |
| AC-CAP-005 | 分类内唯一 | 同分类重名返回 400 + CAP_4002 |
| AC-CAP-006 | 同步能力只读 | 编辑/删除 source=sync 返回 400 + CAP_4004 |
| AC-CAP-007 | 改类型清空配置 | 修改 capability_type 后 request_config 被清空 |
| AC-CAP-008 | 软删除生效 | 删除后 is_deleted=true，默认列表不可见 |
| AC-CAP-009 | 模糊搜索 | 搜索大小写不敏感，包含匹配生效 |
| AC-CAP-010 | 组合筛选 | 按类型+分类筛选，结果全部满足条件 |

## 4. HTTP 配置验收项（P0）
| AC-ID | 场景 | 验收标准 |
|---|---|---|
| AC-HTTP-001 | 方法白名单 | TRACE 等非法方法返回 400 + CAP_4001 |
| AC-HTTP-002 | URL 协议校验 | 非 http/https 返回 400 + CAP_4006 |
| AC-HTTP-003 | 内网地址拦截 | localhost/127.0.0.1/10.x 拦截，返回 400 + CAP_4006 |
| AC-HTTP-004 | 超时范围校验 | 0、负数、61 返回 400 + CAP_4001 |
| AC-HTTP-005 | Header 黑名单 | Host/Content-Length 返回 400 + CAP_4001 |
| AC-HTTP-006 | 成功判定 | 2xx 判成功，4xx/5xx 判失败并落日志 |

## 5. 同步任务验收项（P0）
| AC-ID | 场景 | 验收标准 |
|---|---|---|
| AC-SYNC-001 | 增量同步入库 | 问学新增后本地新增，source=sync |
| AC-SYNC-002 | 同步删除映射 | 问学删除后本地 is_deleted=true |
| AC-SYNC-003 | 失败重试 | 失败重试 3 次，间隔 1s/2s/4s |
| AC-SYNC-004 | 冲突覆盖 | 同步字段按问学值覆盖 |
| AC-SYNC-005 | 审计日志 | 有任务 ID、成功数、失败数、失败原因 |

## 6. 权限与并发验收项（P0）
| AC-ID | 场景 | 验收标准 |
|---|---|---|
| AC-AUTH-001 | 只读角色限制 | 新增/编辑/删除返回 403 |
| AC-AUTH-002 | 运营删除分类限制 | 删除分类返回 403 |
| AC-CON-001 | 乐观锁冲突 | 并发更新后提交方返回 409 + CAP_4091 |
| AC-CON-002 | 删除幂等 | 连续删除同一能力无副作用 |

## 7. 日志与安全验收项（P1）
| AC-ID | 场景 | 验收标准 |
|---|---|---|
| AC-LOG-001 | 操作审计完整 | 日志包含操作者、时间、前后值 |
| AC-LOG-002 | 敏感字段脱敏 | 响应和日志不出现明文 token/密钥 |
| AC-OBS-001 | 调用链追踪 | 外部调用日志包含 trace_id、耗时、状态码 |

## 8. 自动化执行要求
1. API 自动化覆盖全部 P0 用例，并校验错误码与 schema。
2. UI 自动化覆盖分类和能力主路径（新增、编辑、删除、筛选、搜索）。
3. 同步任务使用 mock 服务验证重试和冲突覆盖。
4. 性能测试在 CI 固定数据集执行并输出 P95 报告。
5. 安全测试在 CI 执行 URL 黑名单、越权、敏感信息扫描。
